MyPeptidePrice API ingestion v2 patch

Purpose
Fix missing or incorrectly grouped GLP listings by normalizing all vendor products once on the server before storing the browser snapshot.

Changed files
- netlify/functions/_shared/product-normalizer.mjs
  New shared normalization module. Recognizes plain RETA, SEMA, TIRZ, SM, TZ, TRZ, and RT aliases, plus known coded vendor aliases. Keeps GLP blends separate. Preserves raw listing fields for debugging.

- netlify/functions/all-products.js
  Routes every vendor row through the shared normalizer. Adds ingestion-v2 diagnostics. Adds fresh vendor bypass mode for scheduled refreshes.

- netlify/functions/glow-products.js
  Loads WooCommerce variations for variable products. Preserves individual size, price, and stock rows such as RETA 10mg.

- netlify/functions/flawless-products.js
  Loads WooCommerce variations for variable products. Preserves individual size, price, and stock rows such as RETA 10mg.

- netlify/functions/ion-products.js
  Adds feed metadata so diagnostics report source product counts, variable products, and returned variation rows.

- netlify/functions/refresh-products-snapshot.mjs
  Stores ingestion-v2 snapshots under a new key. Uses a last-known-good safeguard when a vendor feed fails, returns zero rows, or drops suspiciously below its prior count.

- netlify/functions/products-snapshot.mjs
  Reads the ingestion-v2 snapshot key. Uses shorter CDN freshness windows and a schema-specific fallback URL.

- netlify/functions/snapshot-diagnostics.mjs
  New read-only diagnostics endpoint.

- index.html
  Uses the ingestion-v2 snapshot endpoint and localStorage key. Adds plain GLP shorthand recognition to the browser fallback logic. Keeps raw listing names searchable.

- glp-weight-loss.html
  Uses the ingestion-v2 snapshot endpoint and localStorage key. Adds plain GLP shorthand recognition and raw listing search support.

- package.json
  Declares ESM module mode and preserves the Node 20 engine requirement.

- .nvmrc
  Pins Netlify builds to Node 20.

- .npmrc
  Uses the public npm registry.

After deployment
1. Wait for Netlify to publish successfully.
2. Open Logs & metrics, then Functions.
3. Open refresh-products-snapshot and click Run now once.
4. Open /.netlify/functions/products-snapshot?schema=ingestion-v2 and confirm the response includes schema_version: ingestion-v2.
5. Open /.netlify/functions/snapshot-diagnostics and review feed counts, unmapped_glp_candidates, upstream_errors, and preserved_vendors.
6. Verify the homepage Reta filter includes Ion Peptide, Glow Aminos, and Flawless Compounds where their live APIs return eligible rows.
