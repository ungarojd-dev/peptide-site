MyPeptidePrice catalog engine v4

Purpose
- Rebuild the sorting and card grouping system without replacing the stable API pipeline.
- Normalize vendor naming differences in one browser-side engine.
- Ensure a failed optional enrichment feed cannot blank or replace the stable catalog.

Safety design
- Existing stable vendor API functions were not modified.
- Existing products-snapshot and refresh-products-snapshot functions were not modified.
- The homepage still loads the stable cached snapshot first.
- Glow and Flawless variation enrichment loads afterward as an optional overlay.
- If the overlay fails, the stable snapshot remains visible.
- Static fallback rows remain available when a live snapshot is partial.

Engine behavior
- Plain RETA, SEMA, TIRZ, RT, SM, TZ, and TRZ labels are normalized.
- Coded vendor aliases such as ION-3R, PEP-RT, PEP-SM, and PEP-TRZ are normalized.
- Blends remain separate cards.
- Popular filters include relevant blends where appropriate.
- Live rows replace matching static vendor-family rows only.
- Unmatched static families stay visible as a safety fallback.

Internal QA page
- /catalog-debug.html
- Read-only page showing snapshot rows, enrichment rows, rendered offers, card count, vendor counts, and uncategorized rows.
